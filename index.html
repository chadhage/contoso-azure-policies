Five pillars, one baseline

Enterprise-scale management group hierarchy

Guardrails assign at management-group scope so every current and future subscription inherits compliance.

Inheritance model

  • Broad, safe guardrails (locations, tags, audits) assign at contoso.
  • Strict enforcement (Deny, DeployIfNotExists) assigns lower in the tree.
  • Sandbox gets cost controls with audit-only security.
  • Decommissioned denies all new resource creation.

Deploy in three steps

Portal, CLI, or CI/CD. The orchestrator creates definitions, initiatives, and scoped assignments.

  1. 1

    Authenticate

    Sign in and target the Contoso intermediate root management group.

  2. 2

    Deploy the hierarchy

    Optionally stand up the management-group tree with Bicep (tenant scope).

  3. 3

    Apply the baseline

    Run the orchestrator to create definitions, initiatives, and assignments.

# 1. Authenticate
az login
az account set --subscription "<platform-mgmt-sub-id>"

# 2. Deploy the management group hierarchy (tenant-scoped)
az deployment tenant create `
  --name contoso-mg-hierarchy `
  --location eastus `
  --template-file management-groups/hierarchy.bicep

# 3. Deploy definitions, initiatives & assignments
./scripts/deploy.ps1 -RootManagementGroupId contoso -Location eastus -WhatIf
# 1. Authenticate

az deployment tenant create \

Rollout model: onboard in Audit → measure 2–4 weeks → remediate → promote the assignment effect to Deny → time-box any exemptions.
